使用场景3：现在你已经查看过了过去的历史，现在想恢复至目前所在分支 (e.g., master) 上，该使用什么命令？

我第一想法是输入 git checkout HEAD (注意: HEAD 代表了最近的分支的 commit number) 但是该命令的回复却是让我很意外，每一行之前的 M 代表了什么呢？我孤陋寡闻，没有见过这种输出形式啊！猜想应该代表 Modified. 而且我发现，我的代码(文本)还是历史状态，并没有恢复至最近的 commit 版本。

根据我对 git reset --hard HEAD 命令的体会。git checkout <COMMIT_HASH_NUMBER> . 将过去的 working copy 找了出来。 所以，现在每个文件都处于修改并且added的状态(等待commit, 问题：我是否需要git commit一下呢？)。 因为我只是想查看过去某个状态下的代码，不需要把这些历史内容保存下来。所以我觉得可以git reset --hard一下 之所以说 git reset --hard这个命令比较危险。是因为这个命令会讲所有在stagged/unstagged area的数据都清理掉。换言之，所有没有commit的数据都会丢失。当然，如果你很清楚的知道，这些还没有committed 的数据不需要，那么git reset --hard 对你来说是安全无害的。

Linux/MAC OSX环境下如何将Android设备挂载为外设？

发表于 2020-06-23 更新于 2021-02-02

问题

由于安卓设备普遍采用MTP文件系统，而Linux/MAC OSX的文件系统与此不同，所以将安卓设备通过USB连接到电脑上无法被后者识别。因为不能方便地采用诸如rsync等工具来同步手机中的视频招聘到电脑上。 # 分析： 解决方案都只能是在两种不同的文件系统之间做适配。 这个链接：https://www.v2ex.com/t/533974, 做了一些很有启发性的讨论。 # 方案：

Android File Transfer via USB

手机和电脑通过USB模式直连，USB连接至于MTP模式，通过使用谷歌官方推出的Android File Transfer这个程序，访问进安卓设备内部存放照片视频目录，通过复制粘贴的方式，通过USB数据线传输完成文件备份。

ADB (Android Debug Bridge)

官方推荐，也是最快的方式是通过ADB工具实现安卓设备和电脑主机之间的文件互传。如果你电脑上安装有Android Studio，那么你电脑中已经安装有ADB了。一旦你激活了安卓手机上的开发者选项，并且通过ADB命令连接上你的安卓设备。那么在Android Studio中，你可以通过选项卡View > Tool Windows > Device File Explorer实现安卓设备挂载。

sshfs

首先，需要确保MAC OSX系统中已经安装了OSXFUSE以及sshfs。

其次需要在手机上安装sshelper app并进行相应配置，以保证可以通过ssh访问安卓设备。之后在你的电脑上合适的位置创建使用sshfs所需的挂载点。最后运行下列命令完成挂载。 https://askubuntu.com/questions/893090/how-do-i-mount-my-androids-filesystem-over-the-network-with-sshfs

在Hexo博客中插入图片的各种方式

在Hexo博客中插入图片的各种方式

绝对路径本地引用

当Hexo项目中只用到少量图片时，可以将图片统一放在source/images文件夹中，通过markdown语法访问它们。图片既可以在首页内容中访问到，也可以在文章正文中访问到。

相对路径本地引用

图片除了可以放在统一的images文件夹中，还可以放在文章自己的目录中。文章的目录可以通过站点配置文件_config.yml来生成。将_config.yml文件中的配置项post_asset_folder设为true后，执行命令hexo new post_name，在source/_posts中会生成文章post_name.md和同名文件夹post_name。将图片资源放在post_name中，文章就可以使用相对路径引用图片资源了。

注意：
1. 使用markdown语法插入的图片不能设置图片大小; 2. 在hexo较新的版本中，已经不需要插件hexo-asset-image就能支持文章资源文件夹了。安装此插件反而会导致路径出错（图片路径类似于/.com//docker-cnm.png）。因此需要卸载该插件以防引起冲突。

标签插件语法引用

这种相对路径的图片显示方法在博文详情页面显示没有问题，但是在首页预览页面图片将显示不出来。如果希望图片在文章和首页中同时显示，可以使用标签插件语法。

CDN引用

除了在本地存储图片，还可以将图片上传到一些免费的CDN服务中。比如Cloudinary提供的图片CDN服务，在Cloudinary中上传图片后，会生成对应的url地址，将地址直接拿来引用即可。

如果安装完毕，启动网站的时候出现跟hexo-inject相关的错误的时候，可以试着先删除hexo-math,然后再重新安装.

更换Markdown渲染引擎

Hexo的默认渲染器hexo-renderer-marked，不支持Latex语法，一些Latex中的符号会被误认为是markdown语法，比如\、_和*，这让Mathjax不能正确地渲染（Marked.js和MathJax共存时可能会发生一些渲染冲突，而katex需要渲染器支持，根本不能工作）。举个例子，如下面的LaTeX源码，在采用Hexo的默认渲染器且不处理冲突时便会导致公式内的换行符(\\)失效。

这是Next用户的一大痛点，Hexo官方网站对此有比较全面的探讨。为了解决上述问题，社区行成了以下几种思路：

  1. 最简单粗暴的办法是，写Latex公式时，对冲突符号添加markdown转义符号\，这个方法很糟糕，公式书写不便，可移植性也差，不建议考虑。
  2. 修改渲染器代码(比如hexo-renderer-marked中的Marked.js文件)里的正则表达式，改动语法规则，例如去掉markdown里用*强调的语法，参考这篇文章，显然这不是一个好的解决方案。
  3. 更换在markdown渲染器。对于Mathjax，可以选择hexo-renderer-pandoc或hexo-renderer-kramed。
    • hexo-renderer-pandoc用的是pandoc渲染器. pandoc是一种特别设计的markdown语法，用它写的文档可以转换成五花八门的格式，该语法天然支持Latex。这也意味着pandoc与markdown语法不全相同，写文章的时候需要注意。需要指出的是，要使用hexo-renderer-pandoc，则电脑首先需要安装pandoc. 如果电脑采用的是OSX系统以及Homebrew, 则可以通过brew install pandoc来安装。
    • hexo-renderer-kramed引擎是在默认的渲染引擎hexo-renderer-marked的基础上修改了一些bug，两者比较接近，也比较轻量级。但该渲染器也有缺点：不支持行内Latex公式。解决办法是有的——和上面一样，要么在行内自己加上转义符号，要么修改渲染规则。渲染器作者建议是用把公式标注成代码块，参见此处。

修改主题配置文件

hexo把数学渲染相关的配置交由各个主题中的配置文件管理了。我采用的是经典的next主题，是可以直接配置公式支持，可选择mathjax和katex两种主流插件。因此，需要修改该主题目录下的_config.yaml文件。

注意：无论采用mathjax还是katex，在写每篇博文的时候，都需要Front-matter部分还是要加上语句：mathjax: true.否则，数学公式无法被正常渲染.
